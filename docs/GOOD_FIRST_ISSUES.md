# Good First Issues 候选列表

本文档列出了适合新手贡献的任务，按难度分类。

---

## ⭐ 初级（预计 30 分钟 - 1 小时）

### 1. 添加更多 MUD 服务器配置示例

**难度**：⭐ 初级
**文件**：`examples/custom_login.tt`
**描述**：当前只有 3 个 MUD 服务器的登录示例（书剑、西游记、侠客行），添加更多常见的 MUD 服务器配置。

**如何做**：
1. 研究目标 MUD 的连接方式
2. 参考 `custom_login.tt` 中的现有格式
3. 添加新的登录别名（如 `login_xymud`）
4. 更新帮助文本

**资源**：
- [MUD Connect](https://www.mudconnect.com/) - MUD 服务器列表
- 现有示例代码

---

### 2. 补充 README 的常见问题

**难度**：⭐ 初级
**文件**：`README.md`
**描述**：README 中只有 4 个常见问题，补充更多玩家可能遇到的问题。

**候选问题**：
- 如何修改 MUD 服务器地址？
- 地图数据保存在哪里？
- 可以同时连接多个 MUD 吗？
- 如何备份我的地图数据？
- 遇到编码问题怎么办？

**如何做**：
1. 在 `README.md` 的 `常见问题` 部分添加新条目
2. 保持格式一致
3. 如果有解决步骤，给出具体命令

---

### 3. 添加基础战斗别名示例

**难度**：⭐ 初级
**文件**：`examples/basic_usage.tt` 或新文件 `examples/combat.tt`
**描述**：`basic_usage.tt` 只有基础的 `kill` 和 `flee`，添加更多战斗相关别名。

**建议添加**：
- `attack <目标>` - 攻击目标
- `flee <方向>` - 向指定方向逃跑
- `use_item <物品>` - 使用物品
- `check_status` - 检查战斗状态

**如何做**：
1. 创建新别名或扩展现有文件
2. 添加详细注释
3. 提供使用示例

---

### 4. 修复文档中的错别字

**难度**：⭐ 初级
**文件**：`README.md`, `examples/README.md`
**描述**：检查并修复文档中的错别字、格式问题。

**如何做**：
1. 通读所有 Markdown 文件
2. 检查错别字、标点符号
3. 检查链接是否有效
4. 统一格式（如标题层级、列表格式）

---

### 5. 添加英文版 README

**难度**：⭐ 初级
**文件**：`README_EN.md`
**描述**：创建英文版的 README，方便国际用户。

**如何做**：
1. 翻译现有 `README.md`
2. 保留代码示例不变
3. 适当调整文化差异的表达
4. 在主 README 添加语言切换链接

---

## ⭐⭐ 中级（预计 1-2 小时）

### 6. 添加地图数据验证功能

**难度**：⭐⭐ 中级
**文件**：`map_data/map_save.py` 或新文件 `map_data/validate.py`
**描述**：创建一个脚本验证地图数据的完整性。

**功能需求**：
- 检查孤立房间（没有出口连接）
- 检查单向通道（A→B 但 B 没有→A）
- 检查房间命名一致性
- 生成验证报告

**如何做**：
1. 读取 `map_data/raw/` 下的数据
2. 构建图结构
3. 遍历检查异常
4. 输出报告

---

### 7. 添加配置文件模板

**难度**：⭐⭐ 中级
**文件**：新文件 `config_template.tt`
**描述**：创建一个可定制的配置模板，让用户快速配置常用选项。

**功能需求**：
- MUD 服务器地址配置
- 登录方式选择
- 地图采集开关
- 日志级别设置
- 个人别名定义

**如何做**：
1. 设计配置结构
2. 添加详细注释
3. 提供默认值
4. 在 README 中说明使用方法

---

### 8. 添加简单的 GUI 封装

**难度**：⭐⭐ 中级
**文件**：新文件 `start_gui.sh` 或 `start_gui.py`
**描述**：创建一个简单的图形界面，让用户可以一键启动、停止、连接 MUD。

**技术选型**：
- macOS：AppleScript + Terminal
- 跨平台：Python + tkinter/PyQt

**功能需求**：
- 启动/停止按钮
- 状态显示
- 日志查看
- 快捷命令按钮

---

### 9. 添加地图数据导出功能

**难度**：⭐⭐ 中级
**文件**：`map_data/map_save.py`
**描述**：扩展 `map_save.py`，支持更多导出格式。

**建议格式**：
- Graphviz DOT 格式（可视化）
- Markdown 表格格式
- JSON 压缩格式
- CSV 格式

**如何做**：
1. 分析现有导出逻辑
2. 添加新格式支持
3. 添加命令行参数
4. 更新文档

---

## ⭐⭐⭐ 高级（预计 2-4 小时）

### 10. 添加自动探索功能

**难度**：⭐⭐⭐ 高级
**文件**：`examples/auto_explore.tt`
**描述**：实现自动探索未知区域的功能。

**功能需求**：
- 识别未探索的出口
- 自动移动到未知房间
- 避免危险区域
- 记录探索路径
- 遇到障碍时的处理

**挑战**：
- 状态管理（当前位置、目标）
- 循环检测
- 异常处理（战斗、死亡）

---

### 11. 添加战斗自动化系统

**难度**：⭐⭐⭐ 高级
**文件**：`examples/combot.tt`
**描述**：实现自动战斗功能。

**功能需求**：
- 自动攻击敌人
- 血量监控（自动逃跑/吃药）
- 技能释放序列
- 战利品拾取
- 死亡处理

**挑战**：
- 实时状态监控
- 多敌人优先级
- 复杂触发链

---

### 12. 添加地图数据 Web 可视化

**难度**：⭐⭐⭐ 高级
**文件**：新文件 `web_viewer/`
**描述**：创建一个 Web 界面展示地图数据。

**技术栈**：
- 后端：Python Flask/FastAPI
- 前端：HTML + JavaScript + 可视化库（D3.js/Vis.js）

**功能需求**：
- 房间节点展示
- 连接线展示
- 搜索功能
- 路径高亮
- 缩放/拖动

## 📝 如何认领任务

1. 查看 [Issues](https://github.com/dr4g00n/drag00n-just4fun/issues) 是否已有对应 Issue
2. 如果没有，创建新 Issue 并引用本文档
3. 在 Issue 中声明 "我想做这个"
4. 等待维护者确认后开始工作
5. 完成后提交 PR

---

## 💡 贡献提示

- **不懂就问**：在 Discussions 或 Issue 中提问
- **小步提交**：把大任务拆分成小 PR
- **测试优先**：确保你的更改不破坏现有功能
- **文档更新**：代码变更时同步更新文档
